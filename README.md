# Задачи ТЮФ / ТЮЕ

Веб-приложение для совместной работы над задачами научных турниров **ТЮФ**, **ТЮЕ** и **конкурса капитанов**.

Идеи раньше копились в Telegram-чате и раз в год вручную выбирались на турнир. Этот сервис заменяет хаос чата структурированной базой: статусы, канбан-доски, формулировки, комментарии и выгрузка.

## Возможности

- Карточки задач с условием, автором, источниками и датой сообщения в Telegram
- Три доски по назначению: **ТЮЕ**, **ТЮФ и ТЮЕ**, **конкурс капитанов**
- Канбан со статусами жизненного цикла задачи
- Отдельные поля формулировок: черновик → перед отправкой в методкомиссию → итоговая в турнире
- Метаданные участия: турнир, год, номер задачи (для КК — этап и год)
- Комментарии от нескольких авторов
- История изменений (кто / когда / было → стало)
- Вложения: файлы к условию задачи и к комментариям (до 15 МБ)
- Автоназвание: кнопка «Сгенерировать» (DeepSeek, если настроен ключ)
- Экспорт в TXT и CSV (полный и по фильтру)
- Вход по логину/паролю; пароли в БД только как хеши

## Жизненный цикл задачи

1. Сообщение в Telegram  
2. Формулировка перед отправкой  
3. Отправлена в методкомиссию *(для доски «ТЮФ и ТЮЕ»)*  
4. Играется в турнире  
5. Отклонена  

Перетаскивание карточки на канбане меняет статус; при переходе к формулировке или «играется» открывается форма с нужными полями. При откате назад спрашивается подтверждение, если данные будут удалены.

## Как устроены пароли

- В браузере вы вводите логин и пароль.
- В базе (`users`) хранится **не пароль**, а **хеш** (необратимая «каша»). Даже открыв файл БД, нельзя увидеть чужой пароль.
- Смена пароля — в **Настройках** после входа. Меняется только ваш хеш.
- Переменные `PASS_*` в `.env` нужны **только при первом создании** пользователей. Дальше `.env` паролями не управляет.

Логины по умолчанию: `nikita`, `artem`, `ilya`.

## Локальный запуск

```bash
cp .env.example .env
# задайте SECRET_KEY и при желании начальные PASS_*
chmod +x run.sh
./run.sh
```

Приложение: http://127.0.0.1:8000  

Требования: Python 3.10+.

## Как сделать «как надо»: один сервер для всех

Цель: база лежит **на сервере**, все заходят **через браузер** по ссылке, ничего не скачивают.

1. Заведите небольшой VPS или PaaS (**Railway**, **Render**, **Fly.io** и т.п.).
2. Задеплойте этот репозиторий (Docker или `uvicorn app.main:app --host 0.0.0.0 --port $PORT`).
3. В переменных окружения задайте сильный `SECRET_KEY`.
4. Сохраните файл SQLite на **постоянном диске** (volume), либо укажите `DATABASE_URL` на Postgres.
5. Откройте выданный URL → логин/пароль → работаете.
6. Сразу смените пароли в Настройках.

Пока сервера нет, можно по-прежнему запускать локально; для команды удобнее один общий URL.

### Docker (кратко)

```bash
docker build -t tyuf-tyue-zadachi .
docker run -p 8000:8000 \
  -e SECRET_KEY='длинная-случайная-строка' \
  -v "$(pwd)/data:/app/data" \
  tyuf-tyue-zadachi
```
